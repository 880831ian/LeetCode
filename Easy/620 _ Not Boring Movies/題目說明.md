## 不無聊的電影

Table: `Cinema`

```
+----------------+----------+
| Column Name    | Type     |
+----------------+----------+
| id             | int      |
| movie          | varchar  |
| description    | varchar  |
| rating         | float    |
+----------------+----------+
id 是這個表的主 Key	。
每行包含有關電影名稱、類型和評分的訊息。
rating 是小數點後 2 位，在 [ 0,10 ] 範圍內浮動。
```

<br>

寫一個 SQL 查詢來顯示 ID 是奇數且描述不是 `"無聊"` 的電影。

顯示時按 `rating` 降序排序的方式顯示。

查詢結果格式如下例所示。


<br>

**範例 1:**

```
輸入: 
Cinema table:
+----+------------+-------------+--------+
| id | movie      | description | rating |
+----+------------+-------------+--------+
| 1  | War        | great 3D    | 8.9    |
| 2  | Science    | fiction     | 8.5    |
| 3  | irish      | boring      | 6.2    |
| 4  | Ice song   | Fantacy     | 8.6    |
| 5  | House card | Interesting | 9.1    |
+----+------------+-------------+--------+
輸出: 
+----+------------+-------------+--------+
| id | movie      | description | rating |
+----+------------+-------------+--------+
| 5  | House card | Interesting | 9.1    |
| 1  | War        | great 3D    | 8.9    |
+----+------------+-------------+--------+
說明: 
我們有三部電影，奇數 ID 1、3 和 5 。ID = 3 的電影很無聊。因此我們不把他包含在答案中。
```
